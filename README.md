# claude-wordstat

Скилл для **Claude Code** (и просто CLI-клиент на Python) к официальному API
**Яндекс Вордстата** — реальная частотность поисковых запросов, сезонность,
распределение по регионам и сборка SEO-ядра.

Работает на стандартной библиотеке Python (для выгрузки в Excel нужен `openpyxl`).

## Возможности

| Команда | Что делает |
|---|---|
| `top` | популярные + похожие запросы по фразе |
| `dynamics` | динамика частотности во времени (сезонность) |
| `regions` | распределение по регионам + индекс интереса (affinityIndex) |
| `regions-tree` | справочник регионов (ID и названия) |
| `quota` | остаток дневной квоты |
| `core` | батч по списку сид-фраз → агрегированное ядро (`--clean` — очистка и кластеризация) |
| `research` | `top`+`dynamics`+`regions` по фразе → **один .xlsx с вкладками** |

## Установка

### Как скилл Claude Code

Склонируй репозиторий в папку скиллов (имя папки = имя скилла):

```bash
git clone https://github.com/Artemleadl/claude-wordstat ~/.claude/skills/yandex-wordstat
```

После этого Claude Code подхватит скилл `yandex-wordstat` автоматически (он
вызывается на задачах про SEO-ядро, частотность, ключи для Директа и т.п.).

### Как обычный CLI

```bash
git clone https://github.com/Artemleadl/claude-wordstat
cd claude-wordstat
python3 scripts/wordstat.py quota        # проверка (нужен токен, см. ниже)
pip install openpyxl                     # опционально, для --xlsx / research
```

## Получение токена Вордстата

API требует OAuth-токен в заголовке `Authorization: Bearer <token>`.
Делается один раз:

1. Авторизуйся в Яндекс ID: <https://passport.yandex.ru/auth>
2. Создай приложение: <https://oauth.yandex.ru/?dialog=create-client-entry>
   - тип — **«Для доступа к API или отладки»**;
   - в разделе **«Доступ к данным»** включи **«Использование API Вордстата»**;
   - создай приложение, скопируй **ClientId** и **OAuth-токен**.
3. Подай заявку на доступ к API: обратись в поддержку Яндекс Директа, сообщи свой
   логин и `ClientId`. Доступ откроют после одобрения.
4. Положи токен в переменную окружения (**не** в код, **не** в git):

   ```bash
   export WORDSTAT_OAUTH_TOKEN="y0_AgAAA..."
   ```

   Для постоянного хранения добавь строку в `~/.zshrc` / `~/.bashrc`.

Документация API: <https://yandex.com/support2/wordstat/ru/content/api-wordstat>

> ⚠️ Никогда не коммить токен в репозиторий и не вставляй его в сообщения/issue.
> Скрипт читает его только из переменной окружения `WORDSTAT_OAUTH_TOKEN`.

## Использование

```bash
SK=scripts/wordstat.py   # или ~/.claude/skills/yandex-wordstat/scripts/wordstat.py

# Топ + похожие запросы (Москва, мобайл)
python3 "$SK" top "купить кроссовки" --regions 213 --devices phone --num 200

# Сразу в CSV / Excel
python3 "$SK" top "ремонт квартир под ключ" --regions 225 --xlsx keys.xlsx

# Сезонность
python3 "$SK" dynamics "купить елку" --period monthly

# Распределение по регионам + affinity
python3 "$SK" regions "доставка пиццы"

# Справочники
python3 "$SK" regions-tree
python3 "$SK" quota

# SEO-ядро по списку сид-фраз (одна на строку), с очисткой и кластеризацией
python3 "$SK" core seeds.txt --regions 225 --num 500 --clean --xlsx seo_core.xlsx
python3 "$SK" core seeds.txt --clean --min 20 --stop лида деньги   # тонкая настройка

# Единый отчёт по фразе -> один xlsx с вкладками top/dynamics/regions
python3 "$SK" research "лидогенерация" --regions 225 --num 200
```

Без `--csv`/`--xlsx` вывод идёт в stdout как JSON.
`research` сохраняет файл в `results/<фраза>_<дата>.xlsx` (папка создаётся сама).

## Операторы Вордстата

Передавай прямо в тексте фразы:

- `"фраза"` — только эта фраза без хвостов (фиксирует число слов);
- `!слово` — точная словоформа (без склонений);
- `[ ]` — фиксируют порядок слов;
- `+слово` — учитывать стоп-слово (предлоги и т.п.);
- `-слово` — минус-слово (исключить).

Пример точной частотности: `"!купить !iphone 15"`.

## Полезные ID регионов

`225` — Россия, `213` — Москва, `1` — Москва и область, `2` — Санкт-Петербург,
`187` — Украина, `159` — Казахстан, `149` — Беларусь. Полный список — `regions-tree`.

## Лимиты

- Дефолтная квота: **10 запросов/сек**, **~1000 запросов/сутки**.
- На `429` (личная квота) и `503` (общая) клиент сам ждёт и повторяет запрос.
