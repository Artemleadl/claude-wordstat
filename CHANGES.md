## 2026-06-02 — `core --clean`: встроенная очистка и кластеризация

### Изменённые файлы
- `scripts/wordstat.py`, `SKILL.md`

### Что изменено
- В команду `core` добавлены флаги `--clean`, `--min N`, `--stop w1 w2 …`.
- Новые функции: `_words`, `_common_prefix`, `_seed_roots`, `_match_root`,
  `clean_cluster`; константа `RU_STOP`.
- При `--clean`: пул фильтруется по релевантности **самим сид-фразам** (совпадение
  по корню слова с учётом морфологии через общий префикс), добавляется колонка
  `cluster` (= сид-корень). Мелкие кластеры (<3 фраз) → «прочее».
- Доменно-нейтрально: правила берутся из сидов, а не из зашитого словаря темы.

### Проверка (факт, на ядре лидогенерации, 15 сидов, регион 225)
- `py_compile` → OK
- `core --clean` → 789 → **633 фразы, 17 кластеров**; «led»/«кто такой» отсеяны
- `core --clean --min 20 --stop лида деньги заговор привлечь` → **502 фразы, 13 кластеров**,
  город Лида исключён (проверка `'лида' in words` → False)
- Регресс mock-тестов: 15 ok, 0 fail

### Известные ограничения
- Морфо-дубли кластеров («лидогенерация» и «лидогенерации» — разные кластеры).
- Омонимы, совпавшие с корнем сида (город Лида из сида «стоимость лида»), убираются
  только через `--stop`. Для хирургической чистки — ручная дочистка вывода.

## 2026-06-02 — фикс команды `dynamics`

### Изменённые файлы
- `scripts/wordstat.py`

### Что изменено
1. **`dynamics()`** — теперь принимает и отправляет `fromDate`/`toDate` (формат `YYYY-MM-DD`).
   API `/v1/dynamics` требует эти параметры обязательно.
2. **`_default_range(period)`** — новый хелпер: дефолтное окно дат под период с учётом
   выравнивания, которое требует API:
   - `monthly`: `fromDate` = 1-е число месяца, `toDate` = последний день месяца (окно ~12 мес.);
   - `weekly`: неделя Пн…Вс — `fromDate` = понедельник, `toDate` = воскресенье (окно ~26 нед.);
   - `daily`: последние 60 дней, без выравнивания.
3. **`cmd_dynamics`** — использует `--from`/`--to`, иначе берёт дефолтное окно; печатает
   выбранный период в stderr.
4. **CLI `dynamics`** — добавлены аргументы `--from YYYY-MM-DD` и `--to YYYY-MM-DD`.

### Причина
Команда падала с `HTTP 400: fromDate parameter is missing`. Эмпирически (прямыми запросами
к API) выяснены ограничения формата/выравнивания дат и захардкожены в дефолтах.

### Проверка (факт)
- `python3 -m py_compile scripts/wordstat.py` → OK
- `dynamics --period monthly` → 11 точек (2025-06-01 → 2026-04-01)
- `dynamics --period weekly` → 25 точек (2025-12-01 → 2026-05-18)
- `dynamics --period daily` → 56 точек (2026-04-03 → 2026-05-28)
- `dynamics --from 2024-06-01 --to 2025-05-31` → 12 точек
- CSV-экспорт `--csv` → валидный файл с шапкой `phrase,date,count,share`
- Оффлайн mock-тесты: 15 ok, 0 fail (регресс не сломан)
