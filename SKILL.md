# Yandex Wordstat

Скилл даёт доступ к реальной статистике поисковых запросов Яндекса, чтобы любые
материалы для трафика создавались на данных о настоящем спросе, а не вслепую.

## Когда вызывать

- **SEO-ядро / семантика** — собрать частотные и похожие запросы по сид-фразам,
  кластеризовать, отсеять мусор и минус-слова.
- **Статьи / контент под поиск** — понять, по каким формулировкам и с какой
  частотностью люди ищут тему, какие подтемы (похожие запросы) раскрыть.
- **Excel для Яндекс Директа** — собрать ключи + точную частотность в таблицу под
  кампанию. (Заливку объявлений этот скилл пока не делает — только подготовку.)
- **Лендинги, квизы, любые материалы для трафика** — формулировать заголовки,
  офферы и вопросы под реальные запросы аудитории.
- **Проверка спроса / сезонности** — динамика частотности во времени, разбивка
  по регионам и индекс интереса (affinityIndex).

## Авторизация (нативный Wordstat API)

API требует OAuth-токен в заголовке `Authorization: Bearer <token>`.

Как получить (делается один раз):

1. Авторизуйся в Яндекс ID: https://passport.yandex.ru/auth
2. Создай приложение: https://oauth.yandex.ru/?dialog=create-client-entry
   → выбери **«Для доступа к API или отладки»**.
   → в разделе **«Доступ к данным»** включи **«Использование API Вордстата»**.
   → создай приложение и скопируй **ClientId** и **OAuth-токен**.
3. Подай заявку на доступ к API: обратись в поддержку Яндекс Директа, сообщи
   свой логин и `ClientId`. Доступ откроют после одобрения.
4. Положи токен в переменную окружения:

   ```bash
   export WORDSTAT_OAUTH_TOKEN="y0_AgAAA..."
   ```

   Для постоянного хранения добавь строку в `~/.zshrc` / `~/.bashrc`.

Документация: https://yandex.com/support2/wordstat/ru/content/api-wordstat

## Как пользоваться

Клиент — `scripts/wordstat.py` (только stdlib; для `--xlsx` нужен `openpyxl`).

### ВАЖНО: где лежит скрипт

Скрипт находится в подпапке `scripts/` **рядом с этим файлом SKILL.md**, то есть
в директории самого скилла — НЕ в текущем проекте. Никогда не запускай
`scripts/wordstat.py` относительно рабочей папки проекта: его там нет.

Перед запуском определи абсолютный путь к директории скилла (она известна при
загрузке скилла — это папка, содержащая данный SKILL.md). Если путь не очевиден,
найди его так:

```bash
SK=$(find "$HOME/.claude" "$HOME/Library/Application Support/Claude" /root/.claude /tmp /var/folders \
     -name wordstat.py -path '*yandex-wordstat*/scripts/*' 2>/dev/null | head -1)
echo "$SK"   # путь к scripts/wordstat.py этого скилла
```

Дальше во всех командах используй `python3 "$SK" ...`.

### Команды

```bash
# Топ + похожие запросы по фразе (Москва, мобайл)
python "$SK" top "купить кроссовки" --regions 213 --devices phone --num 200

# Сразу в CSV / Excel
python "$SK" top "ремонт квартир под ключ" --regions 225 --xlsx keys.xlsx

# Динамика во времени (сезонность)
python "$SK" dynamics "купить елку" --period monthly

# Распределение по регионам + индекс интереса
python "$SK" regions "доставка пиццы"

# Справочник регионов и остаток квоты
python "$SK" regions-tree
python "$SK" quota

# Батч: собрать ядро по списку сид-фраз (одна на строку) -> агрегированная таблица
python "$SK" core seeds.txt --regions 225 --num 500 --xlsx seo_core.xlsx

# То же + авто-очистка от нерелевантных фраз и разбивка по кластерам
python "$SK" core seeds.txt --regions 225 --num 500 --clean --xlsx seo_core.xlsx
# Тонкая настройка очистки: порог частотности и минус-слова
python "$SK" core seeds.txt --regions 225 --clean --min 20 --stop лида деньги
```

`--clean` (только у `core`) фильтрует пул и кластеризует **обобщённо, по самим
сид-фразам** (не по зашитому словарю темы):
- оставляет только фразы, релевантные сидам (совпадение по корню слова с учётом
  морфологии) — отсекает дрейф ассоциаций («led», «кто такой», чужие города);
- добавляет колонку `cluster` = сид-корень, к которому относится фраза;
- `--min N` — выкинуть фразы с частотностью ниже N; `--stop w1 w2 …` — минус-слова.

Очистка намеренно простая и доменно-нейтральная: она не знает про бренды
конкурентов или омонимы (город, совпавший с корнем сида) — это добивается `--stop`.
Для хирургической чистки сложного ядра по-прежнему лучше дочистить вывод вручную.

Без флага `--csv`/`--xlsx` вывод идёт в stdout как JSON — удобно, чтобы Claude
прочитал данные и обработал их дальше (кластеризация, минус-слова, тексты).

### Единый отчёт по фразе: `research`

```bash
# top + сезонность + гео по фразе -> ОДИН xlsx с тремя вкладками
python "$SK" research "лидогенерация" --regions 225 --num 200
# свой путь к файлу / период / устройства
python "$SK" research "купить кроссовки" --period weekly --devices phone --xlsx report.xlsx
```

- Один запрос → **один .xlsx с вкладками** `top`, `dynamics`, `regions`
  (не россыпь файлов).
- По умолчанию файл: `results/<фраза>_<сегодняшняя дата>.xlsx` (папка создаётся
  автоматически; повторный запуск той же фразы в тот же день перезаписывает файл).
  Путь можно задать через `--xlsx`.
- Вкладка `regions` отсортирована по убыванию спроса. Если отдельный метод упал
  (напр. dynamics по дате) — его вкладка пропускается, остальные сохраняются.

Отдельные команды (`top`/`dynamics`/`regions`/`core`) сами файлы **не** создают:
вывод идёт в stdout (JSON) либо в `--csv`/`--xlsx`, если флаг указан.

### Полезные ID регионов

`225` — Россия, `213` — Москва, `1` — Москва и область, `2` — Санкт-Петербург,
`187` — Украина, `159` — Казахстан, `149` — Беларусь. Полный список — `regions-tree`.

## Операторы Вордстата (уточнение запроса)

Передавай прямо в тексте фразы:

- `"фраза"` — кавычки: только эта фраза без хвостов (фиксирует число слов).
- `!слово` — восклицательный знак: точная словоформа (без склонений).
- `[ ]` — квадратные скобки: фиксируют порядок слов.
- `+слово` — учитывать стоп-слово (предлоги и т.п.).
- `-слово` — минус-слово: исключить запросы с этим словом.

Пример точной частотности: `"!купить !iphone 15"`.

## Рекомендуемый порядок работы

1. От задачи определи 3–15 сид-фраз (или возьми их у пользователя).
2. `core` по сид-фразам → собери широкий пул реальных запросов с частотностью.
3. Почисти: убери нерелевантное, выдели минус-слова, сгруппируй в кластеры.
4. По топ-кластерам при необходимости запусти `dynamics` (сезонность) и
   `regions` (где спрос выше).
5. Отдай результат под конкретный артефакт: SEO-ядро (xlsx), структуру статьи,
   таблицу ключей для Директа, тезисы лендинга/квиза.

## Лимиты и ошибки

- Дефолтная квота: **10 запросов/сек**, **1000 запросов/сутки**.
- `429 Quota limit exceeded ... Time to refill: N seconds` — превышена личная
  квота; скрипт сам ждёт N секунд и повторяет.
- `503 Service unavailable` — общая квота API; скрипт делает бэкофф и ретрай.
- В батче (`core`) для больших списков следи за суточным лимитом.
